나만의 PR 웹 페이지 만들기

튜토리얼 소개

이 튜토리얼을 완료하면, 자기 자신을 소개하는 웹 페이지를 직접 만들어 영하나 홈페이지에 게시할 수 있습니다.

코딩을 전혀 몰라도 괜찮습니다. AI 도구(GitHub Copilot)가 대부분의 코드를 생성해 줄 것이고, 여러분은 무엇을 만들고 싶은지 설명하는 데 집중하면 됩니다.

Tip
왼쪽 사이드바를 통해 원하는 챕터로 이동할 수 있습니다. 처음이라면 순서대로 읽는 것을 권장합니다.

홈 / 개발 환경 준비

개발 환경 준비하기

챕터 2

코드를 작성하려면 도구가 필요합니다. 이 챕터에서는 2가지를 설치합니다.

  • Visual Studio Code (VS Code) — 코드 편집기
  • GitHub Copilot — AI 코딩 어시스턴트

2.1 Visual Studio Code 설치

VS Code는 전 세계에서 가장 많이 사용되는 코드 편집기입니다. 메모장과 비슷하지만, 코드를 작성하기에 훨씬 편리한 기능들을 제공합니다.

설치 방법

  1. code.visualstudio.com에 접속합니다.
  2. 본인의 운영체제(Windows / macOS)에 맞는 버전을 다운로드합니다.
  3. 다운로드된 설치 파일을 실행하고, 기본 설정 그대로 설치를 완료합니다.
▶
GIF 예정
VS Code 공식 사이트 접속 → 다운로드 버튼 클릭 → 설치 과정 전체 화면 비디오
설치 확인
설치가 완료되면 VS Code를 실행합니다. Welcome 탭이 열리는 화면이 나타나면 성공입니다.

2.2 VS Code 한국어 설정 (선택)

VS Code는 기본적으로 영어로 되어 있습니다. 한국어로 변경하고 싶다면 아래 절차를 따릅니다.

  1. 좌측 사이드바에서 Extensions 아이콘(블록 모양)을 클릭합니다.
  2. 검색창에 Korean Language Pack을 입력합니다.
  3. Korean Language Pack for Visual Studio Code를 설치합니다.
  4. 우측 하단에 나타나는 Change Language and Restart 버튼을 클릭합니다.
▶
GIF 예정
Extensions 탭에서 Korean Language Pack 검색 → 설치 → 재시작하여 한국어로 변경되는 전체 과정
Info
한국어 설정은 필수가 아닙니다. 개발 관련 자료 대부분이 영어로 되어 있기 때문에, 영어에 익숙해지는 것도 좋은 선택입니다.

2.3 GitHub Copilot 설정

GitHub Copilot은 코드를 작성할 때 AI가 자동으로 코드를 제안해 주는 도구입니다. 대학교 Education 라이선스를 통해 무료로 사용할 수 있습니다.

Step 1: GitHub 계정 생성

  1. github.com에 접속합니다.
  2. Sign up 버튼을 클릭합니다.
  3. 학교 이메일(또는 개인 이메일)로 계정을 생성합니다.
▶
GIF 예정
GitHub 회원가입 페이지에서 이메일 입력 → 비밀번호 설정 → 계정 생성 완료까지의 과정

Step 2: GitHub Education 인증

학생 인증을 완료하면 GitHub Copilot을 무료로 사용할 수 있습니다.

  1. education.github.com/benefits에 접속합니다.
  2. Get student benefits 버튼을 클릭합니다.
  3. 학교 이메일 인증 또는 학생증 사진 업로드로 학생 신분을 인증합니다.
  4. 승인까지 일정 시일이 소요될 수 있습니다.
주의
학생 인증이 승인될 때까지 시간이 걸릴 수 있습니다. 튜토리얼 진행에 차질이 없도록, 이 단계를 가장 먼저 완료해 주세요. 인증이 지연될 경우 디스코드에서 이운지에게 문의하세요.
▶
GIF 예정
GitHub Education 페이지 접속 → 학생 인증 신청 과정 화면 비디오

Step 3: VS Code에 GitHub Copilot 설치

  1. VS Code를 실행합니다.
  2. 좌측 Extensions 아이콘을 클릭합니다.
  3. 검색창에 GitHub Copilot을 입력합니다.
  4. GitHub Copilot 확장을 설치합니다.
  5. 설치 후 우측 하단에 GitHub 로그인 요청이 나타납니다. 로그인합니다.
▶
GIF 예정
VS Code Extensions에서 GitHub Copilot 검색 → 설치 → GitHub 계정 로그인 연동 과정
설치 확인
설치가 정상적으로 완료되면, VS Code 우측 하단 상태바에 Copilot 아이콘이 나타납니다.

2.4 작업 폴더 만들기

코드 파일을 저장할 폴더를 만들어 둡니다.

  1. 바탕화면 또는 원하는 위치에 my-pr-page 폴더를 생성합니다.
  2. VS Code에서 File → Open Folder(파일 → 폴더 열기)를 클릭합니다.
  3. 방금 만든 my-pr-page 폴더를 선택합니다.
▶
GIF 예정
바탕화면에서 폴더 생성 → VS Code에서 Open Folder로 해당 폴더를 여는 과정

2.5 첫 번째 HTML 파일 만들기

환경이 제대로 구성되었는지 확인하기 위해, 간단한 HTML 파일을 만들어 봅니다.

  1. VS Code Explorer 패널에서 새 파일 아이콘을 클릭합니다.
  2. 파일명을 index.html로 입력합니다.
  3. 아래 코드를 입력합니다.
index.html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <title>테스트</title>
</head>
<body>
    <h1>환경 설정 완료!</h1>
</body>
</html>
  1. Ctrl + S (macOS: Cmd + S)로 저장합니다.
  2. 파일을 더블클릭하거나 브라우저로 드래그하여 열어봅니다.
▶
GIF 예정
VS Code에서 index.html 파일 생성 → 코드 입력 → 저장 → 브라우저에서 열어 확인하는 전체 과정

브라우저에 "환경 설정 완료!"가 표시되면 모든 준비가 끝났습니다.

HTML 기초: 웹 페이지의 뼈대

챕터 3

이 챕터에서는 HTML의 핵심 개념만 빠르게 익힙니다. 모든 것을 외울 필요는 없습니다. GitHub Copilot이 코드를 대신 생성해 주기 때문에, 개념만 이해하면 충분합니다.

3.1 Tag(태그)란?

HTML은 태그(tag)라는 것으로 이루어져 있습니다. 태그는 브라우저에게 "이 내용을 이렇게 표시해라"라고 알려주는 명령어입니다.

태그는 항상 여는 태그와 닫는 태그가 한 쌍을 이룹니다.

<h1>이것은 제목입니다</h1>
  • <h1> — 여는 태그 ("여기부터 제목이야")
  • </h1> — 닫는 태그 ("여기까지가 제목이야")
  • 가운데 텍스트 — 실제로 화면에 표시되는 내용

3.3 HTML 문서의 기본 구조

모든 HTML 파일은 아래의 기본 구조를 따릅니다.

기본 구조
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>페이지 제목</title>
</head>
<body>
    <!-- 여기에 실제 보이는 내용을 작성합니다 -->
</body>
</html>
부분역할
<!DOCTYPE html>"이 문서는 HTML5 문서입니다"라는 선언
<html>문서 전체를 감싸는 최상위 태그
<head>문서의 설정 정보 (탭 제목, 문서 인코딩 등). 화면에 직접 보이지 않음
<body>화면에 실제로 표시되는 모든 내용
Tip
이 구조를 외울 필요는 없습니다. GitHub Copilot에게 "HTML 기본 구조 만들어줘"라고 요청하면 자동으로 생성해 줍니다.

3.4 Attribute(속성)

태그에 추가 정보를 넣을 때 속성(attribute)을 사용합니다.

<a href="https://google.com" target="_blank">구글</a>
  • href="https://google.com" — 이동할 주소
  • target="_blank" — 새 탭에서 열기

속성은 항상 여는 태그 안에 작성하며, 속성명="값" 형태를 따릅니다.

3.5 실습: 간단한 자기소개 페이지

지금까지 배운 태그를 조합하여 간단한 자기소개 페이지를 만들어 봅니다.

index.html 파일을 열고 아래 코드로 교체합니다.

index.html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <title>홍길동 - 자기소개</title>
</head>
<body>
    <h1>홍길동</h1>
    <p>컴퓨터공학과 24학번</p>

    <img src="profile.jpg" alt="프로필 사진">

    <h2>관심 분야</h2>
    <ul>
        <li>웹 개발</li>
        <li>인공지능</li>
        <li>게임 개발</li>
    </ul>

    <h2>SNS</h2>
    <a href="https://instagram.com/example">Instagram</a>
</body>
</html>

아래 플레이그라운드에서 직접 실행해 볼 수 있습니다. 이름이나 관심 분야를 바꿔보세요!

CSS 기초: 웹 페이지 꾸미기

챕터 4

HTML만으로는 텍스트와 이미지가 나열될 뿐, 디자인이 적용되지 않습니다. CSS(Cascading Style Sheets)를 사용하면 색상, 글꼴, 레이아웃 등을 자유롭게 조절할 수 있습니다.

4.1 CSS를 적용하는 방법

CSS를 HTML에 적용하는 방법은 여러 가지가 있지만, 이 튜토리얼에서는 가장 실용적인 방법을 사용합니다.

방법: <style> 태그 사용 (Internal CSS)

HTML 파일의 <head> 안에 <style> 태그를 추가합니다.

index.html
<head>
    <meta charset="UTF-8">
    <title>내 페이지</title>
    <style>
        body {
            background-color: #f0f0f0;
            font-family: sans-serif;
        }
        h1 {
            color: #333333;
        }
    </style>
</head>
이 방법을 권장하는 이유
PR 페이지에서는 <style> 태그로 CSS를 HTML 파일 안에 포함하는 방식을 권장합니다. 파일 하나로 관리할 수 있어 제출이 간편합니다.

4.2 CSS의 기본 문법

CSS는 "어떤 요소에 어떤 스타일을 적용할 것인가"를 정의합니다.

선택자 {
    속성: 값;
}

예시:

h1 {
    color: blue;
    font-size: 32px;
    text-align: center;
}
  • 선택자(Selector): h1 → 스타일을 적용할 대상
  • 속성(Property): color, font-size → 변경할 항목
  • 값(Value): blue, 32px → 적용할 구체적인 값

아래 플레이그라운드에서 CSS 값을 바꿔보세요. color를 red로, font-size를 48px로 바꾸면 어떻게 변할까요?

4.3 자주 사용하는 CSS 속성

색상과 배경

body {
    background-color: #1a1a2e;    /* 배경색 */
    color: #ffffff;               /* 글자색 */
}

글꼴과 텍스트

h1 {
    font-size: 36px;              /* 글자 크기 */
    font-weight: bold;            /* 글자 굵기 */
    text-align: center;           /* 정렬: left, center, right */
    line-height: 1.6;             /* 줄 간격 */
}

여백과 간격

div {
    margin: 20px;                 /* 바깥 여백 */
    padding: 16px;                /* 안쪽 여백 */
}

4.4 Selector(선택자) 종류

스타일을 적용할 대상을 지정하는 3가지 기본 방법입니다.

태그 선택자

해당 태그 전부에 적용됩니다.

p {
    color: gray;
}

Class 선택자

특정 그룹에만 적용할 때 사용합니다. HTML에서 class 속성을 부여하고, CSS에서 .으로 선택합니다.

<!-- HTML -->
<p class="highlight">이 문단만 강조됩니다.</p>
<p>이 문단은 일반입니다.</p>
/* CSS */
.highlight {
    color: red;
    font-weight: bold;
}

ID 선택자

하나의 고유한 요소에만 적용할 때 사용합니다. CSS에서 #으로 선택합니다.

<h1 id="main-title">메인 제목</h1>
#main-title {
    font-size: 48px;
}

5.4 Copilot Inline Suggestion (자동 완성)

Copilot Chat 외에도, 코드를 직접 작성할 때 자동으로 코드를 제안해 주는 기능이 있습니다.

  • HTML 파일에서 코드를 작성하기 시작합니다.
  • 입력을 잠시 멈추면, 회색 텍스트로 Copilot의 제안이 나타납니다.
  • Tab 키를 누르면 제안을 수락합니다.
  • Esc 키를 누르면 제안을 무시합니다.

6.5 자주 발생하는 문제와 해결 방법

문제 1: 이미지가 표시되지 않는다

원인: 파일 경로가 잘못되었거나, 파일이 같은 폴더에 없음

해결:

  1. index.html과 이미지 파일이 같은 폴더에 있는지 확인
  2. HTML 코드의 src 값이 실제 파일명과 정확히 일치하는지 확인
<!-- 파일명이 Profile.JPG인 경우 -->
<img src="Profile.JPG" alt="프로필">  ✅
<img src="profile.jpg" alt="프로필">  ❌ (대소문자 불일치)

문제 2: CSS가 적용되지 않는다

원인: <style> 태그가 <head> 바깥에 있거나, 문법 오류

해결: Copilot Chat에 요청합니다.

이 HTML 코드에서 태그가 올바르게 열리고 닫히는지 검사해줘.
문제가 있으면 수정해줘.

문제 3: Copilot이 예상과 다른 코드를 생성한다

해결: 요청을 더 구체적으로 다시 작성하거나, "이 부분만 수정해줘"라고 범위를 좁혀서 요청합니다.

문제 4: 모바일에서 레이아웃이 깨진다

해결: Copilot에게 요청합니다.

이 페이지를 반응형으로 만들어줘.
모바일(화면 너비 600px 이하)에서 카드가 세로로 쌓이도록 해줘.

7.1 제출 전 최종 체크리스트

제출 전에 아래 항목을 다시 한번 확인하세요.

  • 파일 구성 확인: index.html 파일 1개 + 이미지 파일(들)
  • 파일명 확인: 모든 파일명에 한글, 공백이 포함되어 있지 않은지
  • 이미지 표시 확인: 브라우저에서 프로필 사진이 정상적으로 보이는지
  • 링크 동작 확인: SNS 링크를 클릭했을 때 올바른 페이지로 이동하는지
  • 반응형 확인: 브라우저 창 크기를 줄여도 콘텐츠가 잘 보이는지
  • 오탈자 확인: 이름, 학과, 학번 등 기본 정보에 오류가 없는지
